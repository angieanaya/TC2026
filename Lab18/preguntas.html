Q & A

  • ¿Qué importancia tiene AJAX en el desarrollo de RIA's (Rich Internet Applications?
    Por sus siglas, Asynchronous JavaScript and XML permite la creación de aplicaciones web que funcionen de forma asíncrona, es decir, que procesen cualquier solicitud de servidor en segundo plano lo que hace posible el envío y obtención de datos de servidor sin necesidad de cargar toda la página web.

    De esta manera, AJAX se vuelve la herramienta principal para crear Rich Internet Applications, que son aplicaciones web diseñadas para funcionar como aplicaciones de escritorio, ofreciendo una mejor experiencia visual y mayor interactividad. [1]
  • ¿Qué implicaciones de seguridad tiene AJAX? ¿Dónde se deben hacer las validaciones de seguridad, del lado del cliente o del lado del servidor?
    El desarrollo de aplicaciones AJAX hace más vulnerables a las mismas por las tecnologías en las que se basa y su diseño. Se dice que en una aplicación como esta parte de la lógica se mueve del servidor al cliente, potencializando los riesgos, tales como: Datos corruptos, acceso sin autorización, datos malformados y mal rendimiento.

    En el entorno de una aplicación cliente-servidor los datos deben de ser considerados desde un principio como no confiables y se deben tener en cuenta los ataques de sql injection y el cross-scripting, es por eso que las validaciones del lado del servidor son un must para conservar la seguridad de los datos y la aplicación en general, mientras que las validaciones del lado del cliente usualmente se utilizan con el propósito de ofrecer una mejor experiencia al usuario. [2]
  • ¿Qué es JSON?
    JavaScript Object Notation es un formato ligero para la serialización de datos estructurados lo cual lo hace ideal para usar con API, REST y AJAX. Deriva de los Objetos Literales de Javascript y puede representar cuatro tipos primitivos(strings, números, booleanos y valores nulos), así como dos tipos estructurados(objetos y arreglos). JSON se ha convertido en un reemplazo de XML gracias a su similitud con JavaScript. [3]
