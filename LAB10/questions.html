  • ¿Por qué es una buena práctica separar el controlador de la vista?
    Es una buena práctica separar la lógica del sistema de la interfaz porque permite la modificación de componentes sin necesidad de afectar a los demás, más aún permite separar las entradas y salidas del sistema de la parte con la que interactúa el sistema. [1]
  • Aparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?
    • $_SERVER
      Array that contains information like header, routes and script locations.
    • $_COOKIE
      An associative array of variables passed to the current script via HTTP Cookies.
    • $_SESSION
      An associative array containing session variables available to the current script.
    • $_REQUEST
      An associative arraythat by default contains the contents of $_GET, $_POST and $_COOKIE.
    [2]

  • ¿Qué es XSS y cómo se puede prevenir?
    Cross-Site Scripting (XSS) attacks are a type of injection, in which malicious scripts are introduced into otherwise benign and trusted websites.

    Generally an attacker will sedn in the form of a browser side script, to a different and unsuspecting end user.The end user’s browser has no way to know that the script should not be trusted, and will execute the script. Because it thinks the script came from a trusted source, the malicious script can access any cookies, session tokens, or other sensitive information retained by the browser and used with that site. These scripts can even rewrite the content of the HTML page. [4]

    You can prevent XSS attacks by escaping your output using htmlspecialchars() or htmlentities(). Both PHP functions convert problematic characters into HTML entities causing the injected code to be output harmlessly and not rendered but it’s generally recommended to use htmlspecialchar. [5]
