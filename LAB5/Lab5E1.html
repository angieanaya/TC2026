Q & A

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Las validaciones en JavaScript se consideran una buena práctica pues ayudan a brindar al usuario una mejor experiencia y proteger sus datos.
    Si existe un error en el formulario gracias a las validaciones JS se le notifica de manera inmediata al usuario la correción que debe implementar en lugar de tener que esperar una respuesta por parte del servidor. Esto también ayuda a reducir la carga de peticiones del servidor y por lo tanto mejorar el tiempo de respuesta del mismo en otros casos.
  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Existen dos formas, la primera es deshabilitar el javascript dentro del navegador y la segunda es accediendo al código y al buscar el "action" del formulario puedes ubicar a donde se dirige y de esta manera buscar el url de manera directa.
  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    Es una buena práctica porque se validan que los datos sen correctos antes de llegar al servidor y así se evitan hacer demasiadas peticiones erróneas que sólo vuelven mas lento el proceso. Con estas validaciones también se busca proteger los datos de los usuarios al incitar a estos a crear mejores contraseñas y así facilitar el trabajo de proteger sus cuentas, además se busca proteger el sitio como tal ante usuarios maliciosos. El que sea una buena práctica no significa realemente que sea infalible pero su uso provee de mayor seguridad tanto al sitio como al usuario.
